# Requirements Document

## Introduction

This document specifies the requirements for C.O.T.E.ai (Classroom of the Elite), an AI-powered educational platform that provides two distinct learning modes: an Institution Section for classroom-based learning with teacher and student portals, and an Individual Learner Section for self-directed personalized learning with AI-generated roadmaps. The system transforms educational materials into interactive learning experiences through RAG-based AI assistance, gamified assessments, and adaptive content delivery.

## Glossary

- **System**: The complete C.O.T.E.ai application including both Institution and Individual Learner sections
- **Institution_Section**: The classroom-based learning environment with teacher and student portals
- **Individual_Learner_Section**: The personalized learning environment for self-directed learners
- **Teacher_Portal**: Interface for educators to upload materials, track student progress, and manage classrooms
- **Student_Portal**: Interface for learners to access materials, take assessments, and interact with AI assistant
- **Mode_Selector**: UI component allowing users to choose between Institution and Individual Learner modes
- **Session**: A unique identifier representing a classroom or individual learner's workspace
- **Doubt_Assistant**: RAG-powered AI chatbot that answers questions based on uploaded materials
- **Assessment_System**: Gamified evaluation system with three difficulty levels based on Bloom's Taxonomy
- **Flashcard_Service**: AI-powered service generating topic-wise revision summaries
- **Ingestion_Pipeline**: Service that processes PDF documents and stores them in vector database
- **Vector_Database**: ChromaDB storage for semantic search and retrieval
- **XP_System**: Experience points mechanism for gamification and progress tracking
- **Mistakes_Repository**: Storage and tracking system for incorrect answers with AI explanations
- **Learning_Roadmap**: Structured, time-bound plan generated by AI for individual learners
- **Daily_Resource**: Educational content provided to individual learners based on their roadmap
- **Daily_Assessment**: Evaluation activities for individual learners to measure progress
- **Progress_Tracker**: Component monitoring learner advancement toward goals
- **Roadmap_Generator**: AI service creating personalized learning paths from user prompts
- **Goal_Prompt**: User input describing learning objective and timeline
- **Remedial_Plan**: AI-generated diagnostic plan for learners who fail assessments
- **Cooldown_Period**: Time restriction before retrying failed assessments

## Requirements

### Requirement 1: Document Upload and Ingestion

**User Story:** As a teacher, I want to upload PDF educational materials, so that students can access AI-powered learning features based on those materials.

#### Acceptance Criteria

1. WHEN a teacher uploads PDF files, THE System SHALL validate that files have allowed extensions
2. WHEN valid PDFs are uploaded, THE Ingestion_Pipeline SHALL extract text content from the documents
3. WHEN text is extracted, THE System SHALL chunk the content into semantic segments
4. WHEN content is chunked, THE System SHALL generate embeddings using HuggingFace sentence transformers
5. THE System SHALL store document chunks and embeddings in the Vector_Database with session metadata
6. THE System SHALL process ingestion in the background without blocking the upload response
7. WHEN ingestion completes, THE System SHALL make the content available for retrieval queries

### Requirement 2: RAG-Based Doubt Assistant

**User Story:** As a student, I want to ask questions about my learning materials, so that I can get instant, context-aware explanations.

#### Acceptance Criteria

1. WHEN a student submits a query, THE Doubt_Assistant SHALL retrieve relevant document chunks from the Vector_Database using semantic search
2. THE Doubt_Assistant SHALL use maximum marginal relevance search to balance relevance and diversity
3. WHEN relevant chunks are found, THE Doubt_Assistant SHALL provide a formal technical definition from the source material
4. THE Doubt_Assistant SHALL follow formal definitions with intuitive explanations using analogies
5. THE Doubt_Assistant SHALL include application scenarios or brain teasers to promote higher-order thinking
6. WHEN multiple topics are retrieved, THE Doubt_Assistant SHALL explain how concepts fit together
7. THE Doubt_Assistant SHALL support multilingual responses in English, Hindi, Telugu, and Hinglish
8. WHEN responding in Hindi or Telugu, THE Doubt_Assistant SHALL keep technical terms in English while explaining in the native script
9. WHEN no relevant content is found, THE Doubt_Assistant SHALL provide a helpful fallback message

### Requirement 3: Bloom's Taxonomy Assessment System

**User Story:** As a student, I want to take progressive difficulty assessments, so that I can demonstrate mastery at different cognitive levels.

#### Acceptance Criteria

1. THE Assessment_System SHALL provide three difficulty levels: Level 1 (Recall), Level 2 (Application), Level 3 (Creation)
2. WHEN generating Level 1 assessments, THE System SHALL create 10 MCQs focused on definitions and direct facts
3. WHEN generating Level 2 assessments, THE System SHALL create 10 MCQs with scenario-based application questions
4. WHEN generating Level 3 assessments, THE System SHALL create 5 short-answer questions requiring synthesis and evaluation
5. THE Assessment_System SHALL generate questions using AI based on the session's uploaded materials
6. THE System SHALL cache generated assessments to avoid regeneration
7. WHEN a student passes Level 1 with 8+ correct answers, THE System SHALL unlock Level 2
8. WHEN a student passes Level 2 with 7+ correct answers, THE System SHALL unlock Level 3
9. WHEN a student passes Level 3, THE System SHALL mark the chapter as mastered and advance to the next chapter
10. THE System SHALL provide a 10-minute timer for each assessment

### Requirement 4: XP and Gamification System

**User Story:** As a student, I want to earn experience points for completing assessments, so that I feel motivated and can track my achievements.

#### Acceptance Criteria

1. WHEN a student passes Level 1, THE XP_System SHALL award 50-100 XP randomly
2. WHEN a student passes Level 2, THE XP_System SHALL award 100-150 XP randomly
3. WHEN a student passes Level 3, THE XP_System SHALL award 150-200 XP plus a 500 XP chapter completion bonus
4. THE System SHALL display XP gains immediately after assessment submission
5. THE System SHALL maintain a cumulative XP total for each student
6. THE System SHALL allow students to spend XP on hints during assessments
7. WHEN XP is spent, THE System SHALL deduct the amount from the student's total
8. THE System SHALL prevent XP spending if insufficient balance exists

### Requirement 5: Mistakes Repository and Remedial Learning

**User Story:** As a student, I want to review my mistakes with AI-generated explanations, so that I can learn from errors and improve.

#### Acceptance Criteria

1. WHEN a student answers incorrectly, THE System SHALL store the mistake with question, correct answer, and user's answer
2. THE System SHALL generate AI explanations for each mistake
3. THE Mistakes_Repository SHALL allow students to view all their mistakes across sessions
4. THE System SHALL allow students to add personal comments to mistakes for reflection
5. WHEN a student fails an assessment, THE System SHALL generate a Remedial_Plan with diagnosis and practice questions
6. THE Remedial_Plan SHALL classify the gap as Concept Gap, Application Gap, or Overgeneralization
7. THE Remedial_Plan SHALL provide a guided explanation under 100 words
8. THE Remedial_Plan SHALL include one practice question to verify understanding
9. WHEN a student fails an assessment, THE System SHALL impose a Cooldown_Period of 10 minutes before retry

### Requirement 6: Flashcard Generation

**User Story:** As a student, I want AI-generated flashcards for quick revision, so that I can efficiently review key concepts before exams.

#### Acceptance Criteria

1. WHEN a student requests flashcards, THE Flashcard_Service SHALL retrieve all document chunks for the session
2. THE Flashcard_Service SHALL identify all distinct core topics from the materials
3. THE Flashcard_Service SHALL generate concise summaries for each topic (max 120 words)
4. THE System SHALL format flashcards with bullet points and bold text for key terms
5. THE Flashcard_Service SHALL support multilingual flashcards in English, Hindi, Telugu, and Hinglish
6. WHEN generating Hindi flashcards, THE System SHALL use Devanagari script for explanations
7. WHEN generating Telugu flashcards, THE System SHALL use Telugu script for explanations
8. THE System SHALL keep technical terms in English regardless of selected language
9. THE System SHALL cache generated flashcards to avoid regeneration
10. WHEN a student views flashcards, THE System SHALL award XP for engagement

### Requirement 7: Teacher Portal and Analytics

**User Story:** As a teacher, I want to view class-wide analytics and student progress, so that I can identify struggling students and common misconceptions.

#### Acceptance Criteria

1. THE Teacher_Portal SHALL display a list of all available classrooms (sessions)
2. THE Teacher_Portal SHALL show the distribution of students across difficulty levels
3. THE Teacher_Portal SHALL calculate and display the percentage of students stuck at lower levels
4. THE Teacher_Portal SHALL identify and display common mistakes across the class
5. THE Teacher_Portal SHALL show average attempts per difficulty level
6. THE Teacher_Portal SHALL allow teachers to preview all generated assessments organized by chapter and level
7. THE System SHALL allow teachers to upload review documents with assessment focus and student gap guidance
8. WHEN teacher guidance is provided, THE Doubt_Assistant SHALL adjust responses to align with teacher instructions

### Requirement 8: Chapter Progression System

**User Story:** As a student, I want to progress through chapters sequentially, so that I build knowledge systematically.

#### Acceptance Criteria

1. THE System SHALL organize uploaded PDFs as chapters sorted by upload timestamp (oldest first)
2. THE System SHALL track the current chapter index for each student
3. WHEN a student first accesses a session, THE System SHALL start them at chapter 0, level 1
4. WHEN a student masters Level 3, THE System SHALL advance them to the next chapter and reset to Level 1
5. THE System SHALL display the current chapter title and next chapter title to students
6. THE System SHALL calculate chapter deadlines as 5 days from upload time
7. WHEN a student exceeds the chapter deadline, THE System SHALL mark their status as "lagging"
8. THE System SHALL display warning messages with days late when students are lagging

### Requirement 9: Progress Dashboard

**User Story:** As a student, I want to view my learning progress, so that I can understand my current standing and areas for improvement.

#### Acceptance Criteria

1. THE Progress_Tracker SHALL display current XP total and unlocked level
2. THE Progress_Tracker SHALL show the current chapter title and progress through total chapters
3. THE Progress_Tracker SHALL display assessment history with scores and XP gained
4. THE Progress_Tracker SHALL show recent mistakes with options to add comments
5. THE Progress_Tracker SHALL display status indicators (on track, lagging)
6. WHEN a cooldown is active, THE Progress_Tracker SHALL show remaining time before retry
7. THE Progress_Tracker SHALL display weekly improvement metrics

### Requirement 10: Mode Selection and Individual Learner Section

**User Story:** As a user, I want to choose between Institution mode and Individual Learner mode, so that I can access the learning experience that matches my needs.

#### Acceptance Criteria

1. WHEN a user first accesses the application, THE System SHALL display the Mode_Selector with clear options for Institution and Individual Learner modes
2. WHEN a user selects Institution mode, THE System SHALL navigate to the existing teacher/student portal selection
3. WHEN a user selects Individual Learner mode, THE System SHALL navigate to the Individual_Learner_Section
4. THE System SHALL persist the user's mode selection across sessions
5. THE System SHALL provide a way to switch between modes from within the application

### Requirement 11: Individual Learner Roadmap Generation

**User Story:** As an individual learner, I want to enter a learning goal with a deadline, so that I can receive a personalized roadmap to achieve my objective.

#### Acceptance Criteria

1. WHEN a user enters the Individual_Learner_Section for the first time, THE System SHALL display a prompt input interface
2. WHEN a user submits a Goal_Prompt, THE Roadmap_Generator SHALL parse the learning objective and timeline
3. WHEN the Goal_Prompt contains a valid learning goal and deadline, THE Roadmap_Generator SHALL create a structured Learning_Roadmap with daily milestones
4. WHEN the Goal_Prompt lacks a deadline, THE System SHALL request clarification from the user
5. WHEN the Goal_Prompt is ambiguous, THE System SHALL ask follow-up questions to refine the roadmap
6. THE Roadmap_Generator SHALL break down the learning goal into topics, subtopics, and daily learning objectives
7. THE System SHALL display the generated Learning_Roadmap to the user for review and confirmation
8. THE Roadmap_Generator SHALL analyze the complexity of the learning goal and available timeline
9. WHEN the timeline is insufficient for the goal, THE Roadmap_Generator SHALL warn the user and suggest a realistic timeline
10. THE Roadmap_Generator SHALL sequence topics in a logical learning progression considering prerequisite knowledge

### Requirement 12: Individual Learner Daily Resource Delivery

**User Story:** As an individual learner, I want to receive daily learning resources based on my roadmap, so that I can make consistent progress toward my goal.

#### Acceptance Criteria

1. WHEN a new day begins in the learner's timezone, THE System SHALL generate Daily_Resources aligned with the current roadmap milestone
2. THE System SHALL provide multiple types of Daily_Resources including articles, videos, exercises, and reading materials
3. WHEN a learner accesses the Individual_Learner_Section, THE System SHALL display the current day's resources prominently
4. THE System SHALL allow learners to mark Daily_Resources as completed
5. WHEN all Daily_Resources are completed, THE System SHALL update the Progress_Tracker
6. THE System SHALL allow learners to access previous days' resources for review

### Requirement 13: Individual Learner Daily Assessment

**User Story:** As an individual learner, I want to complete daily assessments, so that I can verify my understanding and track my progress.

#### Acceptance Criteria

1. WHEN a learner completes the Daily_Resources, THE System SHALL provide a Daily_Assessment aligned with the day's learning objectives
2. THE Daily_Assessment SHALL include questions of varying difficulty (recall, application, creation)
3. WHEN a learner submits answers, THE System SHALL evaluate responses and provide immediate feedback
4. THE System SHALL store assessment results for progress tracking
5. WHEN a learner answers incorrectly, THE System SHALL provide explanations and store mistakes for review
6. THE System SHALL calculate a daily score and update the Progress_Tracker
7. THE System SHALL adapt difficulty based on learner performance in assessments

### Requirement 14: Individual Learner Progress Tracking

**User Story:** As an individual learner, I want to track my progress toward my learning goal, so that I can stay motivated and adjust my efforts as needed.

#### Acceptance Criteria

1. THE Progress_Tracker SHALL display the number of days completed versus total days in the roadmap
2. THE Progress_Tracker SHALL show the percentage of learning objectives achieved
3. THE Progress_Tracker SHALL display a visual representation of progress (e.g., progress bar, calendar view)
4. WHEN a learner completes daily activities, THE Progress_Tracker SHALL update in real-time
5. THE Progress_Tracker SHALL highlight upcoming milestones and deadlines
6. THE System SHALL provide weekly progress summaries to the learner
7. WHEN a learner falls behind schedule, THE System SHALL suggest adjustments to the roadmap
8. WHEN a learner consistently performs well, THE System SHALL suggest accelerating the roadmap
9. WHEN a learner struggles with topics, THE System SHALL suggest additional resources or extended timelines

### Requirement 15: Individual Learner Roadmap Modification

**User Story:** As an individual learner, I want to modify my learning roadmap, so that I can adapt to changing circumstances or interests.

#### Acceptance Criteria

1. THE System SHALL allow learners to extend or shorten their deadline
2. WHEN a learner requests roadmap modification, THE Roadmap_Generator SHALL regenerate the plan while preserving completed progress
3. THE System SHALL allow learners to add or remove topics from their roadmap
4. WHEN roadmap changes are made, THE System SHALL update Daily_Resources and assessments accordingly
5. THE System SHALL maintain a history of roadmap modifications

### Requirement 16: Individual Learner Dashboard

**User Story:** As an individual learner, I want a personalized dashboard, so that I can quickly access my roadmap, daily tasks, and progress.

#### Acceptance Criteria

1. THE Individual_Learner_Section SHALL display a dashboard with current day's resources, assessments, and progress
2. THE dashboard SHALL show the learner's current position in the roadmap
3. THE dashboard SHALL display upcoming milestones and deadlines
4. THE dashboard SHALL provide quick access to the AI doubt assistant
5. THE dashboard SHALL show recent mistakes and areas needing improvement
6. WHEN a learner has no active roadmap, THE dashboard SHALL prompt them to create one

### Requirement 17: Individual Learner Data Management

**User Story:** As an individual learner, I want my learning data to be saved and accessible, so that I can continue my learning journey across sessions.

#### Acceptance Criteria

1. THE System SHALL create a unique session identifier for each individual learner
2. THE System SHALL persist Learning_Roadmap data across sessions
3. THE System SHALL store all Daily_Resources, assessments, and progress data
4. WHEN a learner returns to the application, THE System SHALL restore their current roadmap state
5. THE System SHALL allow learners to export their progress and roadmap data
6. THE System SHALL maintain data separation between Institution_Section and Individual_Learner_Section

### Requirement 18: Individual Learner Integration with Existing Features

**User Story:** As an individual learner, I want access to existing cote.ai features like the doubt assistant, so that I can get help when needed.

#### Acceptance Criteria

1. THE Individual_Learner_Section SHALL provide access to the C.O.T.E.ai Doubt Assistant
2. WHEN a learner asks questions, THE Doubt_Assistant SHALL use the learner's roadmap context to provide relevant answers
3. THE Individual_Learner_Section SHALL support the mistakes repository feature for tracking errors
4. THE System SHALL allow learners to generate flashcards from their daily resources
5. THE Individual_Learner_Section SHALL maintain the gamification elements (XP system) adapted for individual learning
